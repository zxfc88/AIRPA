# AI RPA智能体 \- 整体功能设计方案

## 一、项目整体定位

一款**基于大模型智能体、对话式生成RPA自动化流程**的轻量化桌面客户端，产品正式命名：AI RPA智能体。

核心宗旨：**自然语言对话 → AI生成Python自动化代码 → 本地可编辑、可运行、可查看日志 → 支持云端市场下载现成流程**。

区别于传统RPA：无需拖拽、无需编程基础，全程AI智能体驱动，同时支持高阶用户自定义开发。

技术底座：**Tauri 2 \+ React \+ TypeScript \+ Rust**（轻量、高性能、可无限扩展技能）
智能体框架：**LangChain \+ LangGraph（Python）**（状态图驱动的多步骤RPA工作流编排）

## 二、整体核心业务闭环

完整产品链路，五大核心模块形成闭环：

1. **AI对话智能体模块**：自然语言生成自动化需求、生成Python RPA代码

2. **代码编辑与本地工程模块**：代码展示、编辑、指定目录保存、工程化管理

3. **本地运行与日志模块**：客户端内运行Python脚本、实时输出过程、启停控制

4. **智能体技能扩展模块**：可无限新增系统/文件/网络/RPA实操技能

5. **云端流程市场模块**：下载公开/私有自动化流程、本地统一管理运行

## 三、详细功能拆解（全量功能清单）

### 3\.1 AI对话智能体核心功能（产品核心）

- **自然语言需求理解**：用户输入口语化需求（如“自动整理桌面文件”“批量处理Excel数据”）

- **智能体代码生成**：大模型自动生成标准化、可直接运行的Python RPA代码

- **流式对话输出**：实时展示思考过程、代码分段输出

- **对话迭代优化**：支持二次修改需求，AI迭代优化代码、修复报错

- **智能体工具调用**：内置Function Calling，可调用客户端所有本地技能

- **需求转工程化代码**：支持生成单脚本、多文件工程、配置文件、依赖文件

- **需求转标准化工程代码**：AI 根据用户需求自动生成**独立项目工程目录**（目录名=自定义项目工程名称），工程内默认生成 **Main\.py 入口主文件**，同时配套生成依赖文件、配置文件、资源目录，形成完整可运行工程，而非零散单文件脚本

- **需求智能解析与应用自启动**：智能体分析用户自然语言需求后，自动识别目标操作载体，优先适配浏览器场景，自动启动对应浏览器应用，无需用户手动操作

- **页面元素智能探索与提取**：启动应用后，智能体自动探索需求所需操作页面/界面元素，精准提取元素关键属性（定位器、文本、坐标、属性参数等），自动沉淀为RPA操作依据，支撑后续自动化流程生成

- **基于Playwright浏览器自动化**：核心依托Playwright框架实现全浏览器自动化操作，兼容多内核浏览器，支持页面跳转、元素点击、输入、表单提交、数据抓取等全场景操作

- **能力可横向扩展**：主体落地浏览器自动化能力，架构预留扩展接口，后期可无缝拓展桌面客户端软件、本地窗口等系统应用自动化操作能力

### 3\.2 代码展示与工程管理功能

- **VSCode同款代码编辑器**：基于Monaco Editor，支持Python语法高亮、代码折叠、格式化、行号

- **实时代码编辑**：AI生成代码后，用户可手动修改、微调逻辑

- **自定义工作目录**：
        

    - 系统弹窗选择本地任意文件夹作为工作空间

    - 所有生成代码、流程文件、资源文件统一保存至指定目录

    - 永久记忆用户工作目录，重启不丢失

- **标准化项目工程管理**：
        智能体根据用户需求自动生成**独立项目工程文件夹**，文件夹名称即为当前自动化任务的项目工程名称，语义化命名、清晰可识别
    
- 每个自动化工程独立隔离，互不干扰，支持多工程并存管理

- 工程内部默认生成**Main\.py** 作为唯一入口启动文件，所有自动化核心逻辑统一收敛至 Main\.py

- 配套自动生成 requirements\.txt、工程说明、配置文件等附属资源，形成标准完整工程结构

- **本地工程管理**：展示当前目录下所有自动化脚本、分类管理

### 3\.3 本地运行与实时日志功能（核心体验）

- **客户端内置运行引擎**：无需打开CMD/终端，一键运行Python脚本

- **指定目录运行**：脚本在用户选定的工作目录内执行，支持读写本地文件资源

- **实时流式日志输出**：逐行展示print日志、程序运行进度、业务输出

- **实时错误捕获**：自动捕获Python报错、异常信息，标红展示

- **运行状态控制**：一键运行、一键强制停止、清空日志

- **运行记录留存**：保存每次运行日志，支持回看排查问题

### 3\.7 Playwright浏览器自动化专属配套功能

专为核心浏览器RPA自动化场景打造，解决Playwright环境配置繁琐、浏览器内核缺失、驱动适配报错、运行环境冲突等行业痛点，实现**全程零手动配置、全自动适配修复**，全方位保障智能体浏览器探索、元素提取、流程运行的稳定性与精准度。

- **Playwright依赖自动自检**：客户端启动、首次触发浏览器自动化需求时，自动扫描本地环境，精准检测Playwright核心库是否安装、版本是否适配当前Python环境，识别依赖缺失、版本冲突、环境异常等问题。

- **全自动依赖安装适配**：检测到缺失或异常依赖时，后台静默执行适配版本的pip安装、更新操作，自动匹配当前生效Python环境，规避多环境依赖错乱、安装失败问题，无需用户手动输入任何命令。

- **多浏览器内核自动部署**：自动检测本地Playwright专属Chromium、Firefox、WebKit内核状态，自动下载、安装缺失内核，无需用户手动配置浏览器驱动，彻底解决传统自动化工具驱动适配繁琐、版本不匹配报错问题。

- **全浏览器自适应兼容**：原生适配Chrome、Edge、Firefox、Safari等主流浏览器，智能体可根据用户需求自动匹配最优浏览器内核，支持自定义运行浏览器、无头/有头可视化模式自由切换，适配各类网页运行场景。

- **浏览器环境智能修复**：内置专属环境修复机制，针对内核损坏、启动失败、页面加载超时、元素捕获异常、进程残留等问题，可自动重置环境、清理残留进程、重装内核，快速修复运行故障。

- **专属运行环境隔离**：将Playwright浏览器自动化运行环境与本地普通Python工程环境完全隔离，专属依赖、浏览器内核独立管理，杜绝与本地其他项目依赖冲突、版本互相干扰的问题，保障RPA流程持续稳定运行。

- **环境状态可视化管控**：客户端内置Playwright专属环境面板，直观展示依赖版本、浏览器内核状态、环境异常日志、修复记录，支持用户一键环境检测、一键自动修复、一键重置环境。

## 四、智能体技术架构（LangChain + LangGraph）

### 4.1 总体架构

智能体核心基于 **LangChain + LangGraph** 构建，以 Python Sidecar 进程运行，Tauri Rust 后端管理其生命周期，前端通过 Tauri 事件获取流式输出。

```
┌─────────────────────────────────────────────────────────────┐
│                    Tauri 桌面客户端                           │
│  ┌──────────┐   ┌──────────────┐   ┌────────────────────┐  │
│  │  React   │──▶│  Rust 后端   │──▶│ Python Sidecar     │  │
│  │  前端    │◀──│  (Tauri)     │◀──│ (LangChain+LangGraph)│  │
│  │          │   │              │   │                    │  │
│  │  UI 交互 │   │ 进程管理     │   │ 智能体核心逻辑     │  │
│  │  状态管理│   │ 文件系统操作 │   │ RPA 代码生成       │  │
│  │  编辑器  │   │ 权限控制     │   │ 工具调用           │  │
│  └──────────┘   └──────────────┘   └────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

通信协议：**stdin/stdout JSON 行协议**，跨平台通用，无需网络端口，天然适配桌面应用场景。

### 4.2 LangGraph 智能体工作流（StateGraph）

RPA 自动化流程的生成过程建模为 LangGraph StateGraph，包含 5 个核心节点和条件路由：

```
                  ┌──────────────┐
                  │   用户输入    │
                  └──────┬───────┘
                         │
                  ┌──────▼───────┐
                  │ 需求理解      │ understand_requirement
                  │ 解析用户意图  │ 提取关键需求、目标应用、操作步骤
                  └──────┬───────┘
                         │
            ┌────────────┼────────────┐
            │ 需要浏览器? │            │ 不需要浏览器
            └────────────┘            │
                  │                   │
       ┌──────────▼──────────┐        │
       │ 元素探索             │        │
       │ 启动浏览器/探索页面  │        │
       │ 提取元素定位器属性   │        │
       └──────────┬──────────┘        │
                  │                   │
                  └────────┬──────────┘
                           │
                  ┌────────▼─────────┐
                  │ 代码生成          │ generate_code
                  │ 生成标准化Python  │ Main.py + requirements.txt
                  │ RPA自动化代码     │
                  └────────┬─────────┘
                           │
                  ┌────────▼─────────┐
                  │ 代码审查          │ review_code
                  │ 检查语法/安全/    │
                  │ API正确性         │
                  └────────┬─────────┘
                           │
            ┌──────────────┼──────────────┐
            │ 需要修改?     │              │ 通过
            └──────────────┘              │
                  │                       │
                  │ (循环回代码生成)       │
                  │              ┌────────▼─────────┐
                  │              │ 应用到工程        │ apply_to_project
                  │              │ 写入本地工程目录  │
                  │              │ 更新编辑器        │
                  │              └────────┬─────────┘
                  │                       │
                  └───────────────────────┘
                                          │
                                 ┌────────▼─────────┐
                                 │  完成             │
                                 └──────────────────┘
```

**节点职责说明**：

| 节点 | 功能 | 输入 | 输出 |
|------|------|------|------|
| `understand_requirement` | 解析自然语言需求，提取结构化意图 | 用户消息 | 需求描述 {intent, target_app, steps} |
| `element_exploration` | 条件节点：需求涉及浏览器时启动Playwright探索页面 | 需求 + URL | 页面元素列表 [{selector, type, label}] |
| `generate_code` | 生成标准化Python RPA工程代码 | 需求 + 元素 + 上下文 | 工程代码 {main_py, requirements_txt} |
| `review_code` | 审查代码质量，决定是否需要重新生成 | 代码 | 审查结果 {passed, issues, suggestions} |
| `apply_to_project` | 将通过的代码写入本地工程目录 | 最终代码 | 文件写入确认 |

**AgentState 核心数据结构**：

```python
class AgentState(TypedDict):
    messages: Annotated[list, add_messages]  # 对话消息（自动追加）
    requirement: Optional[dict]              # 结构化需求 {intent, target_app, steps, constraints}
    needs_browser: bool                      # 是否需要浏览器自动化
    elements: Optional[List[dict]]           # 页面元素 [{selector, type, label, attributes}]
    generated_code: Optional[dict]           # 工程代码 {main_py, requirements_txt, config_json}
    review_result: Optional[dict]            # 审查结果 {passed, issues, suggestions}
    retry_count: int                         # 迭代次数（防止无限循环）
    current_stage: str                       # 当前工作流阶段
```

### 4.3 LangChain Tools — 技能系统

所有本地技能通过 LangChain `@tool` 装饰器实现，自动注册到智能体，由 LLM 根据用户需求自主决策调用。技能分类如下：

| 技能类别 | 工具函数 | 功能描述 |
|----------|----------|----------|
| **浏览器自动化** | `browser_navigate`、`browser_extract_elements`、`browser_execute_action` | 页面导航、元素探索提取、点击/输入操作 |
| **系统操作** | `file_operation`、`directory_list` | 文件移动/复制/删除、目录遍历 |
| **办公文件** | `excel_read`、`excel_write`、`pdf_read` | Excel/PDF 读写与数据处理 |
| **网络请求** | `http_request`、`api_call` | HTTP 请求、API 调用、数据爬取 |
| **命令执行** | `shell_execute`、`pip_install` | 系统命令执行、Python 依赖安装 |
| **媒体处理** | `screenshot`、`ocr_recognize` | 截图、OCR 文字识别 |

新技能只需编写函数并添加 `@tool` 装饰器即可即插即用，智能体自动识别调用。

### 4.4 智能体流式交互阶段

用户可见的完整交互过程，每个阶段在对话面板实时展示：

1. **"正在分析需求..."** — `understand_requirement` 节点解析用户意图
2. **"正在探索页面元素..."** — `element_exploration` 节点启动浏览器（条件触发）
3. **"正在生成RPA代码..."** — `generate_code` 节点逐块流式输出 Python 代码
4. **"正在审查代码..."** — `review_code` 节点检查代码质量
5. **"代码已生成，请确认是否应用到工程"** — `apply_to_project` 等待用户确认

```txt
# Python Sidecar 核心依赖
```

### 4.6 架构优势总结

1. **状态图驱动**：LangGraph StateGraph 天然适配多步骤 RPA 工作流，流程可观测、可调试、可恢复
2. **技能即工具**：LangChain `@tool` 装饰器与设计方案技能系统完美映射，即插即用无侵入
3. **流式全链路**：LLM Token → LangGraph 节点 → Sidecar stdout → Tauri Event → 前端 UI，端到端流式体验
4. **轻量高性能**：Tauri 架构无冗余内核，对比 Electron 体积缩小 10 倍，内存更低、启动秒开
5. **能力无上限**：智能体技能可无限扩展，适配所有桌面 RPA 场景
6. **工程化规范**：统一工作目录、标准化流程包，支持个人/团队工程管理
7. **体验原生IDE级**：Monaco编辑器+实时日志，媲美专业开发工具
8. **云端本地联动**：自主生成+市场下载双模式，兼顾灵活创作与快速复用
